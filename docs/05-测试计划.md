# 测试计划 — StudyBuddy

## 1. 测试策略
采用四层测试金字塔：
- 单元测试：覆盖核心算法（评分引擎、坐姿检测算法）
- 集成测试：模块间接口（摄像头→坐姿检测→语音提醒）
- 系统测试：端到端场景（开始→写作业→结束→生成视频）
- 验收测试：家长和孩子实际使用场景

## 2. 测试用例清单

### TC-01 坐姿检测
| ID | 场景 | 预期结果 |
|----|------|---------|
| TC-01-01 | 标准坐姿 | 指示灯绿色，合规率>95% |
| TC-01-02 | 头部前倾 (>25°) | 3帧后进入 Warning，5秒后语音提醒 |
| TC-01-03 | 趴桌 | 立即进入 Critical，语音提醒 |
| TC-01-04 | 身体倾斜 >10° | Warning 状态，不提醒 |
| TC-01-05 | 快速恢复标准坐姿 | 10帧内恢复正常状态 |

### TC-02 视频录制

## 3. 测试环境
- Windows 10/11 笔记本
- 内置或外接 USB 摄像头
- Python 3.10+
- 充足光照条件

## 4. 测试工具
- pytest：单元测试框架
- coverage.py：代码覆盖率
- 人工验收：家长+孩子实际使用
